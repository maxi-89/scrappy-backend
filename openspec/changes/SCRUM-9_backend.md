# Backend Implementation Plan: SCRUM-9 — Scraping Worker: Fetch and Store Raw Businesses from Google Maps

## Overview

Implement the async scraping worker that fetches business listings from Google Maps for a given category and zone, and persists raw records to the `businesses` table. The worker is triggered as a `BackgroundTask` after a scraping job is created, updates the job status throughout execution (`pending → running → completed / failed`), and stores each scraped business as a raw record linked to the job.

**Layers involved**: Domain · Application (Worker) · Infrastructure · Presentation (Router trigger)

**DDD principles applied**:
- Worker lives in `app/application/workers/` — orchestrates domain logic without touching HTTP or SQLAlchemy directly
- `Business` is a pure domain entity with invariants enforced in `__post_init__`
- Repository interfaces defined in domain; implementations in infrastructure
- Router remains thin: only schedules the background task

```python
from sqlalchemy import update as sa_update

async def update(self, job: ScrapingJob) -> None:
    stmt = (
        sa_update(ScrapingJobORM)
        .where(ScrapingJobORM.id == job.id)
        .values(
            status=job.status,
            records_scraped=job.records_scraped,
            error_message=job.error_message,
            started_at=job.started_at,
            finished_at=job.finished_at,
        )
    )
    await self._session.execute(stmt)
    await self._session.commit()
```

### Step 6: Application Worker — ScrapingWorker

**File**: `app/application/workers/scraping_worker.py`

The worker receives a `session_factory: async_sessionmaker[AsyncSession]` and `google_maps_api_key: str` via constructor. It does NOT receive pre-built repositories — it creates its own sessions for each DB operation because background tasks outlive the HTTP request session.

**Scraping approach**: Official `googlemaps` Python SDK (Places Text Search + Place Details).
- Text Search query: `"{category} in {zone}"`
- Paginate up to 3 pages (60 results max via `next_page_token`)
- `await asyncio.sleep(2)` between pages (Google API requirement)
- For each result, call `gmaps.place(place_id, fields=[...])` for detailed info
- Wrap all synchronous SDK calls in `asyncio.to_thread()`

**Public interface**:

```python
class ScrapingWorker:
    async def run_by_id(self, job_id: str) -> None:
        """Entry point for BackgroundTask. Fetches the job, then runs."""

    async def run(self, job: ScrapingJob) -> None:
        """
        Full lifecycle:
        1. Mark job as running (started_at = now())
        2. Fetch businesses from Google Maps
        3. Save businesses to DB
        4. Mark job as completed (records_scraped = N, finished_at = now())
        On any exception: mark job as failed (error_message = str(exc), finished_at = now())
        Never re-raises — all exceptions are absorbed and recorded.
        """
```

**Internal `_map_to_domain()` rules**:
- Return `None` if `name` is empty (skip malformed entries)
- Use `Decimal(str(value))` for rating/lat/lng, catching `InvalidOperation` → `None`
- Convert empty strings to `None` for optional string fields
- `is_verified = False` always

**Session pattern** — each DB write gets its own session:

```python
async with self._session_factory() as session:
    job_repo = ScrapingJobRepository(session)
    await job_repo.update(job)
```

### Step 8: Router Update — Trigger Worker on POST

**File**: `app/presentation/routers/scraping_jobs_router.py`

Modify `create_scraping_job` to inject `BackgroundTasks` and `ScrapingWorker`:

```python
@router.post("", response_model=ScrapingJobResponse, status_code=202)
async def create_scraping_job(
    payload: CreateScrapingJobRequest,
    background_tasks: BackgroundTasks,
    _: str = Depends(get_admin_key),
    service: ScrapingJobService = Depends(get_scraping_job_service),
    worker: ScrapingWorker = Depends(get_scraping_worker),
) -> ScrapingJobResponse:
    response = await service.create_job(payload)
    background_tasks.add_task(worker.run_by_id, response.id)
    return response
```

The router passes only the `job_id` string to the background task. The worker fetches the full job from the DB itself (`run_by_id`).

## Error Response Format

| Condition | HTTP Status | Handling |
|---|---|---|
| Missing/invalid `X-Admin-Key` | 401 | `get_admin_key` dependency |
| `category` or `zone` invalid | 422 | Pydantic validation |
| Google Maps API key not set | Job `status="failed"` | Worker catches exception |
| Google Maps API quota exceeded | Job `status="failed"` | Worker catches exception |
| DB failure during scraping | Job `status="failed"` | Worker catches exception |
| Job not found (`GET /{id}`) | 404 | Existing router logic |

The worker never re-raises — all exceptions are absorbed and recorded on the job.

| `googlemaps` | `>=4.10.0` | Google Maps Places API client (official SDK) |

**Why `googlemaps` SDK over Playwright/scraping**:
- Playwright requires a full browser in Lambda (~300 MB), exceeding Lambda's 250 MB unzipped limit
- The official SDK is lightweight, reliable, and returns structured JSON
- Web scraping violates Google's ToS; the official API is the correct approach

Install: `uv add googlemaps`

**mypy**: `googlemaps` ships no type stubs. Add to `pyproject.toml`:

```toml
[[tool.mypy.overrides]]
module = "googlemaps"
ignore_missing_imports = true
```

## Notes

1. **Lambda timeout**: Increase Lambda execution timeout to at least 5 minutes (300s), ideally 15 minutes (900s). A full scrape (60 results = 3 pages + 60 detail calls) takes 2–5 minutes.

2. **`asyncio.sleep(2)` between pages**: Google requires a delay before `next_page_token` becomes valid. This is non-negotiable and uses `asyncio.sleep()` (non-blocking).

3. **Session lifecycle**: The worker creates its own sessions via `session_factory`. The request session closes after the 202 response is sent — the worker must not use it.

4. **`ScrapingJob` is mutable**: The worker mutates `job.status`, `job.started_at`, etc. in-place before calling `update()`. `ScrapingJob` must not be `frozen=True`.

5. **No deduplication**: SCRUM-9 stores all returned records without deduplication. If the same business appears in multiple jobs, it will have multiple rows with different `scraping_job_id` values. Deduplication is a future concern.

6. **`place_id` not stored**: Not required by SCRUM-9. Can be added in a future migration if deduplication by `place_id` is needed.

7. **Testing**: Never hit the real Google API in unit tests. Mock `ScrapingWorker._fetch_businesses` directly or mock `googlemaps.Client`.

8. **Cost**: Places Text Search = $0.032/request; Place Details = $0.017/request. Budget ~$3 per job for 60 results.
